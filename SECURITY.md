# Security Policy

## Public release boundary

This repository is a public distribution package. Do not commit client data, personal information, internal paths, screenshots, prototype source files, credentials, tokens, keys, or confidential business documents.

## Reporting a vulnerability

Please open a private security report through the repository's GitHub Security tab when available. Do not include secrets or confidential files in a public issue. If private reporting is unavailable, open an issue containing only a short, non-sensitive description and request a private follow-up.

## Release checks

Before publishing a release, inspect text and filenames for local paths, usernames, emails, client names, project names, credentials, and copied private artifacts. Also verify that generated project outputs and local backups are not part of the repository.
