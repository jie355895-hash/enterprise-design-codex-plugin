# 工业系统跨域融合参考

## 核心原则

工业 SaaS 不应先被简单归类为纯 MES 或纯 LIMS。一个系统可以按模块融合 MES、LIMS、QMS、IIoT、设备管理、仓储、样品、物料、试验和追溯能力。

`knowledge/lims` 与 `knowledge/mes` 是检索和知识组织上的分类，不代表产品必须拆成两个系统，也不代表两个域不能共享模块或数据。

## 融合模式

- 模块融合：同一平台内同时提供 MES 和 LIMS 模块；
- 流程融合：MES 工单触发 LIMS 检测，检测结果回写 MES；
- 数据融合：设备、样品、批次、物料、结果或库存等对象跨域共享。

一个项目可以同时采用多种融合模式。设计确认稿需要标注适用模式、数据主责和哪些内容仍待确认。

## 设计前必须判断

对于每个模块，明确：

- 所属业务域和产品形态
- 数据主系统及数据负责人
- 上游、下游和接口依赖
- 共享业务对象及统一标识
- 数据同步方向、频率和触发方式
- 角色、组织、工厂、实验室和数据范围
- 状态及流程由哪个域负责
- 属于共享能力、MES 主导、LIMS 主导或待确认

## 常见融合对象

样品、物料、批次、工单、试验任务、仪器、设备、检测结果、质量记录、库存、用户和组织可能跨域使用。具体是否共享，必须以当前项目资料或确认稿为依据。

## 防止重复建设

同一个对象或流程不能在 MES 和 LIMS 中各自复制一套而不说明关系。必须补充：

1. 主记录归属和唯一编号
2. 同步方向及触发条件
3. 状态映射和流程主责
4. 冲突处理与重复提交策略
5. 审计追踪和版本关系
6. 接口失败、补偿和人工处理入口

## 工作流要求

当需求涉及多个工业域时，确认稿和 Demo 说明必须包含跨域模块矩阵，至少覆盖：模块、业务域、角色、核心对象、数据主责、上下游、入口、操作权限、状态流程、同步边界、异常处理、依据来源和待确认项。

对象级分析可使用 `industrial-domain-integration-matrix.md`，进一步记录唯一标识、创建方、修改方、同步触发、重复/乱序消息、失败补偿、人工干预和跨系统审计关系。
