# Enterprise Design Codex Plugin

面向 SaaS 工业系统的产品设计工作流插件，从需求调研、PRD 确认到可交互 Demo、评审和版本保存，帮助产品经理快速形成可查看、可操作、可验收的设计结果。

适用场景包括 LIMS、MES、QMS、IoT、设备与资产管理、质量与检测、试验管理、数据大屏和多系统融合模块。

## 核心能力

- 需求类型识别、证据收集、必要澄清和文字确认稿
- 角色、数据范围、入口、权限、状态和流程驱动的页面设计
- 文字确认稿到 PRD，再到交互 Demo 的确认门槛
- 页面内说明 Drawer、统一说明入口和复杂业务解释规则
- 正常提醒与需处理提醒的分类及处理闭环
- LIMS、MES 与其他工业域的组合设计参考
- 视觉探索、交互检查和 UX 评审
- Demo 前检查、Demo 后检查、角色权限检查和可访问性检查
- 业务内容、Demo、原型和技术依赖的独立版本管理
- 选择空白文件夹时自动判断独立根项目或子项目
- 初始化只补充项目控制文件，不覆盖已有业务输出

## UI 参考层

默认 UI 参考 Ant Design 5、Carbon Design System 和 WCAG 2.2，默认 Demo 画布为 1920×1080，使用 24 栅格、8px 基础单位、14px 基础字号和 6px 基础圆角。`ui-ux-pro-max` 用于视觉方向、风格探索和评审。

技术路由：HTML + Tailwind 使用 `daisyUI`；React + Tailwind 使用 `shadcn/ui`；Vue 3 + Tailwind 在没有现有组件体系时使用 `shadcn-vue`。已有项目的品牌、组件体系和已确认 PRD 优先。

## 安装与使用

1. 将本仓库作为 Codex 插件安装，或将插件目录放入 Codex 用户插件目录。
2. 在 Codex 插件列表中选择 **Enterprise Design Codex Plugin**。
3. 为任务选择一个空白项目文件夹；插件会识别独立项目或已有项目下的子项目，并补齐必要控制文件。
4. 直接在对话中描述需求。无需手动修改插件文件，也无需运行同步脚本。

首次产品设计任务按以下顺序执行：需求识别 → 调研与澄清 → 业务分析 → 文字确认稿 → PRD → PRD 确认 → Demo → 说明与评审 → 用户验收 → 版本保存。

## 项目输出结构

初始化后的项目包含四个业务输出目录和一个受保护的控制目录：

```text
业务内容文档/       PRD、业务规则、需求与交付文档
Demo/                可运行或可查看的页面 Demo、截图和说明
原型/                可编辑原型源文件与交互逻辑
技术依赖/            包配置、锁文件和 Demo 所需依赖
CODEX_PLUGIN_DO_NOT_EDIT/  插件控制记录、检查清单和版本索引
```

四个业务目录初始化时不创建无意义的子目录；有实际产出时再建立版本目录。受保护目录不属于产品交付物，不应手动修改或删除。

## 安全与隐私

本仓库只应包含通用规则、模板和公开参考内容。不要提交客户名称、个人姓名、公司内部路径、业务原文、截图、原型源文件、账号、令牌、密钥或项目专属数据。发布前请运行仓库中的敏感信息检查，并确认没有把本地知识库或项目文件复制进仓库。

## 版本策略

已确认输出不原地覆盖。每次变更创建新版本并记录来源、上一版本、变更原因、确认状态和待确认项；回滚也通过创建新版本完成。
