# 项目交付目录

每个产品设计项目统一使用以下结构：

```text
项目目录
├─ 业务内容文档/                  PRD、需求调研、业务规格、截图和交接资料
├─ Demo/                         根据已确认 PRD 生成的页面 Demo、页面说明和 Demo 资源
├─ 原型/                         源原型、交互逻辑和原型导出物
├─ 技术依赖/                      依赖包、锁文件和构建配置
├─ CODEX_PLUGIN_DO_NOT_EDIT/     插件规则、管理记录和程序文件
└─ AGENTS.md                     Codex 规则入口（隐藏）
```

PRD 文件放入 `业务内容文档`，Demo 文件和页面内说明放入 `Demo`，可编辑原型源文件放入 `原型`。文字版确认稿确认后生成 PRD，PRD 确认后生成 Demo。Demo 记录 PRD 来源和版本关系；如有原型，也记录原型来源和版本关系。插件控制文件不能放入产品输出目录，也不能手动删除或修改。

原型变更后必须先对比原型与 Demo，输出差异和修改方案，等待确认后再修改 Demo，并更新版本、原型来源、变更说明和待确认项。

已确认的 Demo 不得直接覆盖。每次修改必须在 `Demo\versions` 创建新版本，旧版本保持不变，新版本确认并评审后才标记为 `current`。

版本控制适用于全部输出物：`业务内容文档`、`Demo`、`原型` 和 `技术依赖`。任何已确认输出物都不能直接覆盖，必须保留旧版本并创建新版本记录。
