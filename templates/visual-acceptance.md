# 视觉与交互验收

- [ ] 已按目标设备确认画布，默认桌面画布为1920×1080
- [ ] 已完成1920、1440、1366或1280宽度的适配检查（按项目适用范围）
- [ ] 页面模式与用户任务匹配：表格、列表、详情、表单、工作台、看板、时间线、监控或大屏
- [ ] 颜色、字体、布局、间距和组件规则来自外部UI标准或项目确认规范
- [ ] 文本没有被遮挡、截断或溢出
- [ ] 组件尺寸稳定，不因状态变化跳动
- [ ] 表格、表单、图表的使用方式合理，核心数据可比较和理解
- [ ] 图表包含标题、单位、时间范围、数据来源、刷新时间和必要阈值
- [ ] 图标、按钮、状态标签和说明入口风格统一
- [ ] 普通文字和关键图形满足对比度要求
- [ ] 键盘焦点可见且不被固定层、弹窗或Drawer遮挡
- [ ] 关键操作目标尺寸满足Web或PDA使用场景
- [ ] 状态不只依赖颜色表达
- [ ] 空状态、加载状态、错误状态、无权限和未保存状态可验证
- [ ] Drawer 可打开、关闭并覆盖正确对象
- [ ] 关键操作有明确反馈和失败恢复路径
- [ ] 角色切换后页面内容符合权限矩阵
